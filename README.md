### jdi
Just Do It

#### It's a game and following are the rules - 

Scope: Only today. After today this file becomes invalid and so does everything in it or attached to it. And the same task cannot be picked up until 3 days later.
A repeating cycle has to have 3 tasks minimum, the maximum limit is left to the user.      
Task: Does not matter. Decide anything. But it can only be 1 thing and then do it.    
Details of the task have to be broken down in a file called do.txt, like activities and expected outcomes.    

#### Criteria
First: Clarity. Everything we do has to have its what, why and hows ready / clear.    
Second: Precise, Finite, Closed not open ended.    
Third: Format or Structured Learning. The task should have some sense of format, organization which helps in record keeping, optimizations,
	so that we don't need to redo the whole thing again and again.     

#### Goal
To overcome my epic procrastination habit and everytime I decide to do something, it very quickly, gets overwhelming as I go over
the list of things I want to do which has a lot of fucking items. So I invented this game for myself, to tie the monkey to a pole so that
it doesn't keep jumping all over the place.
BTW, all of this is happening in office hours ;P
