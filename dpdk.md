| NIC             | For dpdk use, NIC must support - <br/>PCIe not PCI<br/>MSI/MSI-X interrupts<br/>DMA<br/><br/><br/>The motherboard must have IOMMU hardware.<br/>                                                                                                                                                                                                                                          | The kernel should have iommu enabled in boot configuration.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |             |
| IOMMU           | Its the same as MMU but for IO devices.<br/>Needed for direct and safe access to device.<br/>Device DMA → IOMMU → Restricted physical memory region<br/>IOMMU groups - Its a grouping of 2 or more devices which cannot be isolated due to hardware topology constraint. Its like 2 people that live on the same side of the road i.e. PCIe bus.<br/>A group is isolation unit.<br/><br/> | Kernel boot param - <br/><br/>intel_iommu=on<br/><br/>Check kernel has IOMMU enabled - <br/>cat /proc/cmdline <br/>BOOT_IMAGE=/boot/vmlinuz-6.8.0-107-generic root=UUID=e079d9c1-2157-44f0-873a-6add3bf496a7 ro intel_iommu=on iommu=pt default_hugepagesz=1G hugepagesz=1G hugepages=32 transparent_hugepage=never vfio_pci.enable_sriov=1<br/><br/>If not then we need to check BIOS settings.                                                                                                                                                                                                                                                                                                                                                          |             |
| VFIO-PCI        |                                                                                                                                                                                                                                                                                                                                                                                           | Software driver that exposes a PCI device safely to user space via IOMMU.<br/>On recent linux kernels it should be there. Check using **lsmod** if not found run **modprobe vfio_pci**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |             |
| Readiness Check | Hardware is good to go. It does not need anything else other than ethernet cable to connect to peer interface.                                                                                                                                                                                                                                                                            | After establishing that we have all we need from hardware, platform needs to do the following - <br/>1. Bind NIC to vfio_pci driver instead of kernel driver.<br/>2. Reserve huge pages at bootup to avoid runtime allocation failure because by then the memory would have become fragmented and huge pages are physically contiguous which will result in allocation failure and hence application won't start.<br/>3. CPU isolation. This is needed to ensure that we have a processor core reserved exclusively for our application which is done by essentially disabling the kernel scheduler for that core as a result of which kernel scheduler will not schedule any process on this core. This core is practically in-visible to the scheduler. |             |
